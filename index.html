MyHL for Hyperliquid — Privacy Policy

Effective June 24, 2026 · STRIDE LABS INC · riley@stridelabs.co

This policy describes how the MyHL for Hyperliquid Chrome extension (“MyHL”, “we”, “us”) handles information when you use it on app.hyperliquid.xyz.

Scope

MyHL runs only on Hyperliquid’s web app. It does not run on other websites. MyHL is not affiliated with Hyperliquid.

Live data mode (default)

New installs use Live data by default. MyHL contacts our hosted read-only data API at myhl-production.up.railway.app:

What is sentWhenStored by MyHL?
Your public Ethereum wallet address Portfolio / net-exposure features that load your account summary No long-term database. In-memory cache on our server for up to 30 seconds, then discarded.
Public market identifiers (coin symbols) Chart overlay layers you enable (funding, L2 book, diffs, calendar) Public market data only; in-memory cache from 5 seconds to 5 minutes depending on data type, then discarded.

Our server proxies Hyperliquid’s public API to return read-only account and market data to your browser. We use this solely to render overlays and portfolio views you see on the page.

Demo data mode

You can switch to Demo data in the extension brand menu. In Demo mode, MyHL does not contact MyHL servers. Chart overlays and portfolio views use bundled or synthetic sample data shipped inside the extension package. Your public wallet address is not sent to us in Demo mode. An in-product disclosure appears if you switch from Demo back to Live.

Direct requests to Hyperliquid

MyHL may send one optional request directly from your browser to api.hyperliquid.xyz for public market metadata (asset leverage limits) used by the what-if liquidation calculator. That request does not include your public wallet address.

Hyperliquid’s own site traffic (including data Hyperliquid loads for your session) is governed by Hyperliquid’s policies, not this policy.

What we do not collect or transmit

  • Private keys, seed phrases, or signing material
  • Orders you place or trade history uploads
  • Analytics or usage telemetry

How your public wallet address is used

MyHL reads your public Ethereum wallet address from the Hyperliquid page (URL, UI, or Hyperliquid’s own API traffic on the page) to power portfolio views in your browser. In Live data mode (the default), that public address is sent to our API when portfolio or net-exposure features load your account summary. In Demo data mode, your public wallet address is never sent to MyHL servers.

Local storage

Feature toggles, overlay preferences, and your Demo/Live choice are stored in your browser’s localStorage on your device. They are not synced to MyHL servers.

Third parties

  • Hyperliquid — public API used as the data source when Live mode is on.
  • Railway — hosts the MyHL data API. Platform HTTP logs may transiently include request URLs.

Chrome Web Store Limited Use

MyHL’s use of information received from Google APIs or Chrome Web Store user data adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. We do not sell user data. We do not use public wallet or account data for purposes unrelated to MyHL’s core functionality (displaying Hyperliquid portfolio and chart overlays). We do not use this data to determine creditworthiness or for lending.

Security

Live-mode API traffic uses HTTPS. No executable code is downloaded from our servers; only JSON data is fetched.

Changes

We may update this policy when MyHL’s behavior changes. The effective date at the top will be revised. Continued use after an update constitutes acceptance of the revised policy.
